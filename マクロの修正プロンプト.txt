Claude Code用プロンプト（先頭/末尾に挿入・可視セル×選択範囲）








あなたはExcel VBAの熟練開発者です。次の仕様を満たすマクロを作成してください。

目的

Excelで ソート／フィルタ後に表示されているセルのうち、さらに ユーザーが選択している範囲（Selection）に含まれるセルだけを対象にして、各セルの文字列に対して 任意の文字列を「先頭」または「末尾」に追加したい。

実行方法

ユーザーがシート上で対象セル範囲を選択した状態でマクロを実行。

マクロ実行時に UserForm を表示し、

挿入文字列（Prefix/Suffix）

追加位置：先頭 or 末尾（OptionButton か ComboBox）

（可能なら）「すでに同じ文字列が先頭についている場合は二重追加しない」チェックボックス
を入力させる。

OKで実行、Cancelで何もしないで終了。

重要条件（対象セル）

Selection内

かつ 可視セルのみ

フィルタで非表示の行は対象外

手動で非表示にした行も対象外

SpecialCells(xlCellTypeVisible) を使用して可視セルのみ抽出すること

追加ルール

追加位置が「先頭」の場合： X + ABC → XABC

追加位置が「末尾」の場合： ABC + X → ABCX

空白セルは基本スキップ（ただし実装が容易なら「空白も追加する」オプションがあっても良い）

数値/日付など文字列以外も対象とし、.Value を文字列として扱って追加（表示形式を壊さないのが理想。難しければ文字列に変換して上書きでもOKだがコメントで注意点を書く）

「二重追加スキップ」オプションの条件（最重要）

可能なら CheckBox を用意してON/OFFできるようにする：

ONのとき：先頭追加なら「すでに先頭が入力文字列で始まっている場合はスキップ」

末尾追加なら「すでに末尾が入力文字列で終わっている場合はスキップ」

ただし、チェックボックス実装が難しい/省略する場合は、必ず “常に追加（同じ文字列でも追加される）” の挙動にしてほしい。

例：先頭が既に「X」でもさらに「X」を追加して XXABC になるのはOK（＝スキップしない）

安全性・例外処理

可視セルが1つも無い、または Selection が無効な場合はメッセージを出して終了。

SpecialCells のエラーを丁寧に処理（On Error Resume Next → 判定 → On Error GoTo 0）。

速度対策として Application.ScreenUpdating / Calculation / EnableEvents を適切に制御し、エラー時も必ず復帰する構造にする（Finally相当の復帰処理）。

元に戻せないため、処理前に確認ダイアログで「対象セル数」「追加文字列」「先頭/末尾」を表示してYes/No確認する。

実装要件

標準モジュールに Public Sub InsertTextToVisibleSelection() を作成。

UserForm 名：frmInsertText

TextBox：txtTextToAdd

追加位置選択：

OptionButton：optHead（先頭） / optTail（末尾） など

CheckBox（可能なら）：chkSkipDuplicate（二重追加をスキップ）

CommandButton：btnOK / btnCancel

仕上げとして以下を提示すること：

標準モジュールのコード一式

UserForm のコード一式

フォーム部品の配置・名前・初期値（どれをONにするか等）

使い方手順（選択→実行→入力→反映）

備考

結合セルはスキップでOK（エラー回避優先）。

可能なら対象は Selection が複数エリアでも動くようにする。

型式コピープロンプト




Claude Code 用プロンプト（完全版・ミスりにくい仕様固定）

あなたはExcel VBAの熟練者です。以下の仕様でマクロを実装してください。
対象ブックは ThisWorkbook、対象シート名は固定で 「入力シート」。

0. 実行形態（絶対）

常時監視（Worksheet_Change / SelectionChange 等）は一切使わない

ユーザーが マクロ一覧（Alt+F8）から手動実行できる通常のSubのみ

Sub名は必ず 日本語で：

Sub 型式・入線本数コピー()

（日本語Sub名が環境で不安定な場合に備えて、同じ処理を呼ぶ英名ラッパー Sub Katashiki_Nyusen_Copy() を追加してもよいが、日本語Subは必須）

1. 対象範囲

シート「入力シート」

2行目～C列の最終行

C列が空白の行は対象外（スキップ）

2. 目的（更新は“全グループ一致”のときだけ）

C列の文字列が同じ行を1グループとして扱い、全グループが不一致なしであることを確認できた場合のみ、グループ内を以下で統一する：

E列の値

F列の値

G列の値

X列の値

J列の背景色（Interior）

統一の基準（マスター行）は そのC値グループの最初の行（上から順）。

3. 最重要：不一致が1件でもあれば「何も変更しない」
3-1. 不一致判定（事前チェックのみ、ここでは絶対に書き換えない）

同一Cグループ内で、E列またはF列またはG列の値が複数種類存在する場合、そのグループは「不一致」。

空欄も値として扱う（空欄＝1種類）

値の比較は “見た目の同一性” を重視し、以下の規則で正規化して比較：

CStr(cell.Value2) を基本（Null/Emptyは "" に寄せる）

前後スペースは Trimして比較（ただし要件としてスペースが意味を持つならTrimしない。ここは Trimする で固定）

数値/文字の差異（例：1 と "1"）は 同じとみなす（CStrで統一する）

※比較は必ず Value2 ベースにする（Valueは日付等でブレることがある）。

3-2. 不一致があった場合の動作（絶対）

不一致グループが1つでもあれば：

E/F/G/X も J背景色も含めて、シートの内容・書式を一切変更しない

MsgBox で **「不一致C値だけ」**を改行区切りで表示して終了する

重複なし

表示はC値のみ（行番号・列名・詳細は不要）

例：

EFG列の値が一致しないC値があります。
処理を中止しました。

AAA
BBB
CCC

C値が多すぎてMsgBoxが長くなる場合に備え、次の仕様で制限してよい：

最大表示件数 MAX_SHOW = 200

200件を超える場合は末尾に （ほかN件） を付ける

4. 不一致が無い場合のみ更新処理を実行
4-1. 変更してよい範囲（厳守）

変更してよいのは E/F/G/X の値 と J列の背景色（Interior） のみ。
それ以外の列の値・書式は 絶対に変更しない。

4-2. 更新ロジック（マスター行へ統一）

各Cグループについて：

マスター行の E/F/G/X 値を取得し、同グループの全行の E/F/G/X を マスター値へ上書きして統一

同様に J列の背景色を マスター行のJのInteriorへ統一

「塗りつぶし無し」も含めて可能な限り正確に反映するため、最低限：

Interior.Pattern

Interior.Color（PatternがxlNoneでも扱い注意）

Interior.ColorIndex（Colorを使うなら基本不要だが、互換のため適切に）

Interior.PatternColor

Interior.PatternColorIndex

可能なら Interior.TintAndShade も反映（取得/設定できる場合）

5. 実装上の“絶対事故防止”要件（ここが完全版の肝）

二段階構成を徹底

まず全グループを走査して 不一致の有無だけ判定（この段階では一切書き換えない）

不一致がゼロだった場合のみ、別ループで更新する

途中で例外が出てもExcel設定が戻るように、必ず On Error GoTo で復帰処理を書く

Application.ScreenUpdating / Application.EnableEvents / Application.Calculation を制御し、必ず元に戻す

Calculationは現状値を保持して復帰

速度のために Scripting.Dictionary を使用して C値→行番号コレクション を作る

late binding（CreateObject）で参照設定不要にしてもよい

ただし、最終行の取得は安定した方法で：

lastRow = ws.Cells(ws.Rows.Count, "C").End(xlUp).Row

lastRow < 2 のときは「対象なし」で終了

6. 期待する納品物

標準モジュールに以下を作成：

Option Explicit

Public Sub 型式・入線本数コピー()

必要な補助関数（例：C値正規化、値正規化、Interiorコピーなど）

使い方をコメントで：

「Alt+F8 → 型式・入線本数コピー → 実行」

7. 追加（任意だが推奨）

不一致が無く更新が完了した場合は MsgBox "完了しました" 程度の簡易通知を出してよい（必須ではない）

変更件数カウントは不要（今回はエラー表示が主）

この仕様どおりに、**「不一致が1つでもあれば完全にノータッチで終了」**を最優先で実装してください。
不一致判定の段階で誤ってセルを書き換えるコード（例：Interiorの参照を代入しようとして変更が起きる等）は絶対に入れないでください。
